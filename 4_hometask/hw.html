Построение лесов

Прежде всего стоит напомнить, что мы можем использовать случайные леса также и для задачи регрессии. Если в decision tree для классификации мы останавливались в построении дерева, когда в листовой вершине оказывались все объекты одного класса, то в задаче регрессии мы хотим, чтобы в листовой вершине были объекты с “относительно близкими” значениями зависимой переменной, которую мы пытаемся предсказать. Тогда среднее значение зависимой переменной по всем объектам внутри листовой вершины и будет предсказывать значение зависимой переменной для новых объектов.

Например, если бы мы хотели, для iris dataset предсказать зависимость Petal.Width ~ Petal.Length + Sepal.Length + Sepal.Width, мы могли бы использовать analysis of variance (ANOVA) метод, для того, чтобы придумывать правила разбиения нашей тренировочной выборки. Затем мы для листовых вершин посчитали бы среднее значение Petal.Width, и регрессия для нового объекта происходила бы следующим образом: двигаться по правилам в дереве, пока не пришёл в листовую вершину; когда пришёл в листовую вершину – верни среднее значение в этой вершине.

На картинке представлено дерево регрессии, со средним значением Petal.Width в каждой вершине.

Функция-обертка

Первая часть задачи будет заключаться в том, чтобы написать функцию-обертку, которая для обучающей и тестирующей выборок, несколько раз построила случайный лес с определенными параметрами, и взяла среднюю ошибку между всеми запусками ошибку на обучающей выборке и на тренирующей выборке.

Что должна делать функция:

  • Принимать обучающую и тестирующую выборки, и количество запусков N
  • N раз построить randomForest с этими данными
  • Для каждого randomForest почитать RMSE (root mean squared error) для обучающей и тренирующей выборок
  • Посчитать средний RMSE для обоих выборок по всем запускам
  • Вернуть вектор двух чисел: первое число – средний RMSE на обучающей выборке, второе – средний RMSE на тестирующей выборке.

RMSE – root mean square error, пускай \(y_i\) – реальное значение возраста, а \(\hat{y_i}\) – предсказанное значение возраста, тогда: \[ RMSE = \sqrt{MSE(\hat{y})} = \sqrt{\frac{1}{n} SSE(\hat{y})} = \sqrt{\frac{1}{n} \sum_{i=i}^{n}{(\hat{y_i} - y_i)^2}} \]

Шаблон функции примерно такой:

#' randomForest wrapper and error estimator
#'
#' @param train.data data.frame, training dataset
#' @param train.response numeric vector, values of dependent variables in training dataset
#' @param test.data data.frame, testing (validation) dataset
#' @param test.response numeric vector, values of dependent variables in testing dataset
#' @param runs.number numeric (integer), how many times we should run random forest
#' @param ... parameters that are passes to randomForest function, like
#'        ntree, mtry, nodesize, replace, sampsize
#'
#' @return numeric vector with two values, 
#'      first is mean of RMSE values on training data
#'      second is mean of RMSE values on testing data
#' @export
#'
#' @examples
wrapper <- function(train.data, train.response,
                    test.data, test.response, 
                    runs.number=50, ...) {
  # put your code inside
}

Функция должна запускать randomForest runs.number раз, а параметры для randomForest передаются после runs.number. Примеры работы функции, можно посмотреть ниже.

# запуск randomForest с аргументами по умолчанию 50 раз и подсчет средней ошибки
errors.defaults <- wrapper(train.data, train.response, test.data, test.response, 50)
print(errors.defaults)
## [1]  5.059343 11.283348
# запуск randomForest всего с 1 деревом внутри 50 раз и подсчет средней ошики
errors.ntree1 <- wrapper(train.data, train.response, test.data, test.response, 50, ntree=1)
print(errors.ntree1)
## [1]  9.698245 15.846369

Неудивительно, однако случайный лес состоящий из одного дерева предсказывает оба датасета хуже, чем лес, построенный по умолчанию (в нём 500 деревьев).

Оптимизация обучения

Теперь, когда вы написали функцию-обертку можно приступать к обучению random forest с разными параметрами, и смотреть на значения RMSE, которые при таком обучении получаются.

Параметры случайного леса

Мы с вами будем оптимизировать наш случайный лес по нескольким параметрам (эти параметры, являются аргументами функции randomForest). Напомню для сводки, что пускай \(N\) – количество объектов в тренировочном датасете, \(M\) – количество features в нашем датасете.

  • ntree – количество деревьев в случайном лесе, по умолчанию 500
  • replace – когда делается bagging (bootstrapping) нашего случайного леса, должны мы это делать с возвращением, или нет? По умолчанию, мы делает bagging с возвращением.
  • sampsize – когда делается bagging (bootstrapping) нашего случайного леса, сколько мы должны взять объектов из тренировочного датасета? По умолчанию, если replace==TRUE мы берем все \(N\) объектов, а если FALSE, то \(\frac{2}{3} N\)
  • nodesize – минимальный размер (по количеству объектов) для листовых вершин, значение по умолчанию – 5
  • mtry – количество признаков, которое случайно выбирается при каждом разбиении (это также называется feature bagging)

Таким образом, если бы мы хотели, чтобы в нашем лесу, все деревья были переобучены, мы бы запустили это как-нибудь в духе:

# запуск randomForest со всеми вершинами (nodesize=1), replace=F, sampsize=N, mtry=M
errors.overfit <- wrapper(train.data, train.response, test.data, test.response, 50,
                          nodesize=1, replace=F, sampsize=40, mtry=10, ntree=100)
print(errors.overfit)
## [1]  0.00000 17.32558

CROSS VALIDATION

Теперь, когда вы выбрали значения всех параметров для нашего случайного леса, я хочу, чтобы вы сделали кросс-валидацию. Кросс-валидация – это процесс, когда мы разбиваем датасет на части, и по очереди используем каждую часть в качестве тестирующей выборки, а остальное – в качестве тренирующей. Затем берем среднее значение ошибки по всем частям, и выдаем из за результат.

Ниже я покажу, как сделать кросс-валидацию на randomForest со значениями по умолчанию: разобьем всех наших доноров на 5 частей по 10 человек в каждой части, по очереди будем использовать эти части, как тестирующие выборки, а оставшуюся часть датасета как обучающую. Посчитаем ошибку для каждого разбиения и затем среднюю ошибку.

Вам же нужно подставить в randomForest свои параметры, и посмотреть как хорошо работает ваш random forest по сравнению со случайным лесом по умолчанию.

  test.response <- response[test.sample]
  
  # calculating RMSE for every part and default random forest
  return(wrapper(train.data, train.response, test.data, test.response, 100))
